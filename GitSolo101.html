Github Solo 101

When working alone and using GitHub for version control, then following commands will get you through most of what you will be doing.
(it is assumed that you have created an account on github.com, created a github repository and installed git (git bash on Windows) before starting here)

  1. Create a directory on your local PC and cd into that directory.
  2. Type "git init" to create the files Git will use to track your changes.
  3. Create the file(s) that you will include in this repository (files and subdirectories will be part of the repository).
  4. Type "git add -A" to let Git know that you want all files and directories included in the repository.
  5. Type "git commit -m "<Version comments go here>"" to tell Git you are creating a new version with a description (version comments).
  6. Login to GitHub and create a new repository.
    1. Click on your name near the top left corner.
    2. Click on "Repositories" near the top of your screen.
    3. Click the "New" button (green) near the upper right corner.
    4. Enter the repository name and description, select Public or Private and leave the boxes below unchecked to create an empty repository.
    5. click "Create repository".
  7. Now back to your computer.
  8. Type "git remote add origin https://github.com/<your GitHub account name>/<your repository name>" to tell git the repository name to update with your files.
    Note: origin is a variable name (nickname) that can be anything you like. Just make sure you use it later with "push" or "pull" commands.
    Substitute your GitHub account name between the slashes after "github.com" and your GitHub repository name after the last slash.
  9. Type "git branch -M main" to tell git to update the main branch.
  10. type "git push -u origin main" to update the GitHub repository with your (committed) files.
    Be sure to use the same name for "origin" that you used in the "git remote add" command.
  11. Now your GitHub repository should contain a copy of the commited files from your computer.


Notes:
  • To test ssh access to GitHub from Git Bash, type "ssh -T git@github.com" - responds "Hi <your github id>"! You've successfully authenticated, but GitHub does not provide shell access."
    Then type "git fetch" (no output) or "git push -u <origin> <branch>" to confirm that everything is up to date.
  • To change the origin repository use this command: "git remote set-url origin <url>" (where origin is customary nickname"). You can also use a different nickname with the new URL.
