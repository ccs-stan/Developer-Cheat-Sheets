Github Solo 101

When working alone and using GitHub for version control, then following commands will get you through most of what you will be doing.
(it is assumed that you have created an account on github.com and installed git (git bash on Windows) before starting here)

On your Local PC (Git)

  1. Create a directory on your local PC and cd into that directory.
  2. Type "git init" to create the files Git will use to track your changes.
  3. Create the file(s) that you will include in this repository (files and subdirectories will be part of the repository).
  4. Type "git add -A" to let Git know that you want all files and directories included in the repository.
  5. Type "git commit -m "<Version comments go here>"" to tell Git you are creating a new version with a description (version comments).

In the cloud (Github)

  1. Click on your name near the top left corner.
  2. Click on "Repositories" near the top of your screen.
  3. Click the "New" button (green) near the upper right corner.
  4. Enter the repository name and description, select Public or Private and leave the boxes below unchecked to create an empty repository.
  5. click "Create repository".

Now update the cloud with your changes

  1. Now back to your computer.
  2. Type "git remote add origin https://github.com/<your GitHub account name>/<your repository name>" to tell git the repository name to update with your files.
    Note: origin is a variable name that can be anything you like. Just make sure you use it later with "push" or "pull" commands.
    Substitute your GitHub account name between the slashes after "github.com" and your GitHub repository name after the last slash.
  3. Type "git branch -M main" to tell git to update the main branch.
  4. type "git push -u origin main" to update the GitHub repository with your (committed) files.
    Be sure to use the same name for "origin" that you used in the "git remote add" command.
  5. Now your GitHub repository should contain a copy of the commited files from your computer.
