Github 201

Creating and working within a branch. Branches allow you to make code changes to your project while leaving the main branch (production) alone. Here's how you might want to use a Branch:

  • Two bugs have been identified in the code (both bug fixes may take some time so we don't want to make changes in our main (production) branch
  • You've been assigned to fix one of the bugs
  • You will need to create a copy (branch) of the code that you can change
  • You will want to make changes and test your code using the new branch
  • When your changes have been tested and are ready for production use, you'll want to merge your changes into the main (production) branch code

Working on your Local PC (Git/GitBash)

  1. Create a directory on your local PC and cd into that directory.
    • mkdir bugfix
    • cd bugfix
  2. Type "git clone " to create a copy of the production repository for you to changes
  3. Type "cd repo to create put your terminal in the directory with your working copy of the cloned code
  4. Type "git branch bugfix1 to create a branch for you to use to change and test your code updates
  5. Type "git checkout bugfix to align your changes to the branch you created to make your code changes
  6. Make your program changes and test your code until you have approval to merge your changes into the production version of the code (use VS Code)
  7. Type "git add -A the new files you created that will fix the bug
  8. Type "git commit -m "bug fix 1 updates - give details here" log a record of what was changed
  9. Type "git push --set-upstream origin bugfix" to copy your branch to the Github repository
  10. Type "git merge" merges your changes into the local copy of main branch (the merge must also be done on Github - browser)

In the cloud (Github)

  1. Click on "{number} branches" next to where you select your branch
  2. Click on "New pull request" on the line labeled "bugfix" in the "Active branches" area
  3. Click the "Create pull request" button below the comments area - use default comments from your local push
  4. CLick the "Merge pull request",then "Confirm merge" buttons to update the main branch from the bugfix branch
  5. click "Create repository".
  6. You can optionally click "Delete branch" to remove the bugfix branch
